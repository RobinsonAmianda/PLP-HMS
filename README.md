# Hospital Management System (HMS)

Introduction

The Hospital Management System (HMS) is a web-based application designed to manage the day-to-day operations of a hospital. The system allows users to register, login, and access various features such as patient management, appointment scheduling, and billing.

Links

* Frontend: https://plp-hms-1.vercel.app/

* Backend: https://plp-hms.onrender.com/

Features

The HMS application has the following features:

* Patient Management: Users can register new patients, view patient details, and update patient information.

* Appointment Scheduling: Users can schedule appointments for patients and view appointment history.

* Billing: Users can generate bills for patients and view payment history.

* User Management: Admin users can manage user accounts, including creating new users and updating existing user information.

Technical Requirements

The HMS application is built using the MERN (MongoDB, Express.js, React.js, Node.js) stack. The backend API is hosted on Render, while the frontend is hosted on Vercel.

Frontend

The frontend is built using React.js and uses the following dependencies:

* React Router: For client-side routing

* Redux: For state management

* Material-UI: For styling and layout

Backend

The backend is built using Node.js and Express.js, and uses the following dependencies:

* Mongoose: For interacting with the MongoDB database

* Bcrypt: For password hashing and verification

* JWT: For authentication and authorization

Database

The application uses a MongoDB database to store data. The database has the following collections:

* Patients: Stores patient information

* Appointments: Stores appointment information

* Bills: Stores billing information

* Users: Stores user information

API Endpoints

The backend API has the following endpoints:

* GET /api/patients: Retrieves a list of all patients

* POST /api/patients: Creates a new patient

* GET /api/appointments: Retrieves a list of all appointments

* POST /api/appointments: Creates a new appointment

* GET /api/bills: Retrieves a list of all bills

* POST /api/bills: Creates a new bill

* POST /api/login: Authenticates a user

* POST /api/register: Registers a new user

Contributing

Contributions to the project are welcome. To contribute, please fork the repository and submit a pull request with your changes.
